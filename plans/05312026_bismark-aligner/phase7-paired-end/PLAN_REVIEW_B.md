The implementer must compute both length guards in the caller exactly as Perl 3864/3869 (separate `+1` increments of `genomic_sequence_could_not_be_extracted_count`, each `return 0` on first failure → **R1 failure short-circuits and R2 is never checked**, Perl 3867 returns before 3869). **Add an explicit plan sub-step + a unit test for "mate2 at chr-edge, mate1 fine → exactly one could-not-extract count, R1 guard passes."** Without it the count will drift on chr-edge pairs. → **CRITICAL.**
