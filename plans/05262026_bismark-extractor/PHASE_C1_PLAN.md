## 1. Goal

Correct the `--no_overlap` predicate polarity in `rust/bismark-extractor/src/overlap.rs` so the Rust extractor produces byte-identical methylation-call output to Perl `bismark_methylation_extractor` v0.25.1 on PE input. The current implementation drops R2's UNIQUE-region calls and keeps the OVERLAP-region calls — the biological opposite of Perl's behaviour. On the 10M PE WGBS smoke this manifests as a **1.87× call-count gap** across every output file.

In addition: rewrite SPEC §7.4 to reflect the actual Perl logic (which the SPEC rev 2 misread by overlooking the coordinate pre-mutations at `bismark_methylation_extractor:2401` for OT pairs and `:2415-2416` for OB pairs), and update every overlap-related test assertion to verify the correct polarity.

**Out-of-scope (separately tracked):**
- #863 `--parallel N` record ordering — orthogonal byte-identity gap.
- #864 splitting-report format — orthogonal byte-identity gap.
- #865 empty CTOT/CTOB file emission — orthogonal byte-identity gap.
- Performance: Phase H smoke saw 0.8–0.9× speedup, well below SPEC §9.7's 4× target. **Defer**: re-measure after this fix lands because Rust will then emit ~88% more output bytes than it does today, which materially changes the I/O profile.

### 2.2 What the current SPEC says vs what Perl actually does

SPEC §7.4 (current, rev 2) claims:

> *Perl writes the skip predicate as `>=` (forward) and `<=` (reverse). The Rust keep predicate is the inverse: strict `<` (forward) and strict `>` (reverse). Both Phase C reviewers verified this against Perl 2905 + 2989.*

This is wrong on **one substantive count** and one cosmetic one:

1. **Cosmetic — wrong Perl branch cited.** The cited lines `2905` / `2989` live inside the `if ($merge_non_CpG)` 2-context branch. The default 4-context strand-specific output (which the harness exercises) is at lines `3744-3747` / `3825-3828`. The **predicate text** at the two locations is byte-identical (same `>=` for `$strand='+'`, same `<=` for `$strand='-'`) — so the wrong citation didn't mislead about which predicate fires, just about where in the file to find the canonical instance.

2. **Substantive — missed the coordinate transformation.** Both Perl predicates use `$start` and `$end_read_1` **after** the line-2401 / line-2416 mutations. The "natural" leftmost-up iteration of R2 the SPEC modelled does NOT happen. Perl iterates R2 from R2's RIGHTMOST position downward for OT pairs (via `$start = R2_rightmost` + `$start - $index` arithmetic) and from R2's LEFTMOST position upward for OB pairs (via `$start = R2_leftmost` + `$start + $index` arithmetic). In both cases, the iteration enters R1's territory FROM R2's UNIQUE REGION, and `return` fires the moment iteration crosses into the overlap.

The biological semantics, once decoded, are correct: **R2's calls in the overlap region (where R1 already has data) are dropped; R2's calls in R2's unique region are kept.** That's what `--no_overlap` should do per its own docstring (Perl `:5860+`): *"only methylation calls of read 1 are kept for overlapping regions"*. Rust currently implements the opposite.

## 3. Behavior

### 3.1 Functional spec for the corrected `drop_overlap`

Given R2's methylation calls (`Vec<MethCall>`, each with `ref_pos: u32` in 1-based reference coordinates) and the parent `BismarkPair` providing R1's alignment start + CIGAR + the pair strand:

**For OT and CTOB (forward) pairs** — R1 leftmost, R2 downstream:

1. Compute `r1_ref_end = pair.r1().cigar().reference_end(r1_start)` — R1's 1-based inclusive last reference position (already exposed by `bismark-io`, no change).
2. **Keep R2 call iff `c.ref_pos > r1_ref_end`** (strict greater-than). I.e., keep R2 calls whose reference position is strictly past R1's last covered position — R2's downstream-unique region.
3. Drop R2 calls with `c.ref_pos <= r1_ref_end` — the overlap region (positions also covered by R1, which `--no_overlap` deduplicates).

**For OB and CTOT (reverse) pairs** — R2 leftmost, R1 downstream:

1. Compute `r1_ref_start = r1_start` — R1's 1-based first reference position (the BAM `alignment_start` field, no transformation).
2. **Keep R2 call iff `c.ref_pos < r1_ref_start`** (strict less-than). I.e., keep R2 calls whose reference position is strictly before R1's first covered position — R2's upstream-unique region.
3. Drop R2 calls with `c.ref_pos >= r1_ref_start` — the overlap region.

### 3.2 Monotonicity equivalence (`Vec::retain` ≡ Perl early-return)

Perl uses an early-return iteration: `for $index (0..$#methylation_calls) { ... if drop_predicate { return; } emit_call; }`. Rust uses `Vec::retain`, which is a set-based filter. **The two emit the same set if and only if the iteration's `ref_pos` sequence is monotonic in iteration order.**

For Bismark XM strings the invariant holds:
- **OT R2** (strand `-` in dispatch, `$start - $index` arithmetic): `ref_pos` is strictly **decreasing** as `$index` increases.
- **OB R2** (strand `+` in dispatch, `$start + $index` arithmetic): `ref_pos` is strictly **increasing** as `$index` increases.

Both directions are monotonic on `ref_pos`. The Rust `aligned_positions` iterator (in `bismark-io`) yields positions in the same order. Therefore Perl's "return at first overlap-region call" is equivalent to Rust's "retain only unique-region calls" for any well-formed CIGAR. This invariant is inherited from the SAM spec's CIGAR semantics.

### 3.3 Edge cases

| Case | Geometry | Expected behaviour |
|---|---|---|
| Non-overlapping forward pair (R2 wholly past R1) | R1=[a,b], R2=[c,d], b < c | **All R2 calls KEPT** (every r2_pos > r1_ref_end). This is the case the current Rust drops to zero. |
| Non-overlapping reverse pair (R2 wholly before R1) | R2=[a,b], R1=[c,d], b < c | **All R2 calls KEPT** (every r2_pos < r1_ref_start). Symmetric. |
| Fully-overlapping pair (R2 ⊆ R1) | R1=[a,d], R2=[b,c], a ≤ b, c ≤ d | **All R2 calls DROPPED.** R2 has no unique region. |

## 4. Signature

```rust

///
/// Keeps R2 calls in R2's UNIQUE region (past R1 for OT/CTOB, before R1 for
/// OB/CTOT). Drops R2 calls in the overlap region (positions also covered by
/// R1) to prevent double-counting in the methylation summary — the documented
/// semantics of `--no_overlap` ("only methylation calls of read 1 are kept
/// for overlapping regions", per Perl POD at line 5860+).
///
/// Mirrors Perl `bismark_methylation_extractor` lines 3825-3828 (OT/CTOB R2,
/// strand='-' branch in the default 4-context strand-specific output) and
/// 3744-3747 (OB/CTOT R2, strand='+' branch in the same section). The
/// predicates are byte-identical to the corresponding predicates in three
/// other Perl branches (`--comprehensive` at 3576/3657; `--merge_non_CpG` at
/// 2905/2987; `--comprehensive --merge_non_CpG` around 4065); the citations
/// above point at the active code path for the default harness invocation.
///
/// Accounts for Perl's pre-mutation of `$start_read_2` / `$start_read_1` at
/// lines 2398-2402 and 2414-2416 respectively (the iteration direction is
/// "from R2's unique region into R1's territory until the boundary is
/// crossed, then early-return").
///
/// ## Polarity (SPEC §7.4 rev 3)
///
/// | Pair class | Perl drop predicate (post-transformation) | Rust keep predicate |
/// |---|---|---|
/// | OT / CTOB | `r2_pos <= r1_ref_end` (line 3826) | `r2_pos > r1_ref_end` |
/// | OB / CTOT | `r2_pos >= r1_ref_start` (line 3745) | `r2_pos < r1_ref_start` |
///
/// Boundary handling is strict (Perl uses inclusive `<=` / `>=` drop
/// predicates; the Rust keep predicates are the strict inverse).
///
/// ## Monotonicity invariant
///
/// `Vec::retain` is set-based; Perl's iteration is early-return. These
/// produce the same set because `c.ref_pos` is monotonic in the Perl

/// from SAM spec CIGAR semantics.

/// ## Errors

/// `BismarkExtractorError::InternalError` if R1 lacks an `alignment_start`
/// (filtered upstream by `bismark-io::open_reader`'s unmapped-filter, so this
/// should not fire in practice).
pub fn drop_overlap(
    mut r2_calls: Vec<MethCall>,
    pair: &BismarkPair,
) -> Result<Vec<MethCall>, BismarkExtractorError> { … }
```

## 6. Efficiency

- **Complexity:** `Vec::retain` is O(R2_calls) per pair. Unchanged.
- **Memory:** `Vec::retain` modifies in place; no realloc when most elements are kept (the post-fix common case). Pre-fix, the common case was "most elements dropped", which `retain` also handles without realloc but at the cost of one pass + element shifting. Post-fix throughput on real WGBS should be slightly better because fewer shifts on average for partial-overlap pairs.
- **Branch prediction:** the polarity flip changes which side of the predicate is "common" but the branch is still inside `retain`'s closure. Negligible.
- **Alternative considered (per Reviewer B):** `binary_search` + `split_off` would be O(log R2) instead of O(R2). Not worth it for ~150-element vectors typical of R2 lengths. Constant-factor overhead of `binary_search` likely beats the `retain` walk only for very large R2 lengths. Keep `retain`.

### 8.1 From Perl source review (load-bearing — verify during implementation)

- **A1.** `$MDN_count_X` in Perl equals R_X's CIGAR-derived reference span length (sum of M+D+N op-lengths). Confirmed by the comment at line ~2396: *"Matching bases, deletions or skipped regions affect the genomic position of the 3' ends of reads, insertions don't"*.

- **A3.** `MethCall::ref_pos` stores the 1-based reference position of each C call (matching Perl's `$start + $index + $pos_offset` output). Verified by the byte-identity of SE outputs (Rust SE emits the same positions Perl does for every record). Also verified at `rust/bismark-io/src/record.rs:293`.
- **A4.** The Perl no_overlap predicate is **only** applied to R2 (never to R1). Confirmed by lines `2436, 2440, 2444, 2448` (R1 dispatch passes `$no_overlap=0` and `$end_read_1=0`; R2 dispatch passes the live `$no_overlap` and `$end_read_1`). - **A5.** For PE input, Perl always sets `$no_overlap=1` (the default since v0.12.6 per line 1215).
