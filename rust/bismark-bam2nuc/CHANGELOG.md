# Changelog — bismark-bam2nuc

All notable changes to this crate are documented here. Versions follow the
sibling Rust-port convention (`1.0.0-alpha.N` during the initial port).

## 1.0.0-alpha.1 — initial Rust port

Port of Perl `bam2nuc` v0.25.1 (mono-/di-nucleotide coverage QC).

- **Byte-identical to Perl v0.25.1** for `*.nucleotide_stats.txt` and
  `genomic_nucleotide_frequencies.txt`, verified by a local Perl-oracle golden
  suite (`tests/golden.rs`, fixtures generated by `tests/data/generate_goldens.sh`)
  and an oxy real-data gate (`scripts/bam2nuc_byte_identity.sh`).
- Reads BAM via pure-Rust `noodles` (raw `RecordBuf`s — tag-agnostic, no XR/XG/XM
  required); SE/PE auto-detected from the `@PG` header. No samtools subprocess.
- Genome reader shared in spirit with `bismark-coverage2cytosine` (four-suffix glob
  priority, `Mus_musculus.NCBIM37.fa` skip, uppercase-on-load, CRLF strip).
- Allocation-free `[u64; 256]` + packed-`u16` nucleotide counter; `mimalloc` global
  allocator.
- Modes: normal (BAM → stats) and `--genomic_composition_only`.
- Scope: BAM input only; `.sam` and `.cram` are rejected with a clear error
  (`.cram` deferred to a later release).
- Faithfully replicates two latent Perl behaviours required for byte-identity: the
  paired-end `elsif ($flag == 83 or 163)` always-true reverse-complement, and the
  case-sensitive / no-dot-anchor output-name derivation.
